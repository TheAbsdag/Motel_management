# Persistence Schema — V0.1.4.1

**Date**: July 2026 (commit `aa1b232` — current HEAD)  
**Version field**: `"version": 3` in every persistence file  
**Purpose**: Patch release introducing CSV export capability alongside existing XLSX turn reports, and internal JSON schema bump to v3 with tower number standardization and `RoomData` record embedding.  
**Releases**: V0.1.4.1

---

## Overview

- **CSV export**: write-only companion files generated alongside `.xlsx` turn reports. Never read/parsed back by the application.
- **JSON schema v3**: backward-compatible migration from v2; tower numbers corrected from 1-based to 0-based, `RoomData` records embedded in all turn activities.
- **Timezone**: `America/Bogota` — all timestamps use `ZonedDateTime` with this zone.
- **Versioning**: Every file carries `"version": 3`. Automated migration v2 → v3 runs on `TurnDetails.fromJson()`.
- **Data directories**: `data/` (active), `backup/` (timestamped snapshots), `history/` (completed turns), `reports/` (XLSX + CSV exports).

---

## 1. CSV Export Format

**Location**: `reports/Turno_<turnNumber>_<yyyy-MM-dd-HH-mm-ss>.csv`  
**Delimiter**: Semicolon (`;`) — not comma  
**Encoding**: UTF-8  
**Generation**: Automatic companion to every `TurnReportGenerator.generateReport()` call.

### Header block (first 11 lines)

| Key | Value | Always present |
|-----|-------|:---:|
| `TURNO` | Turn number | yes |
| `INICIO` | Turn start datetime | yes |
| `FIN` | Turn end datetime (blank if active) | yes |
| `HABITACIONES` | Total rooms revenue | yes |
| `PRODUCTOS` | Total items revenue | yes |
| `TOTAL VENTAS` | Total sales | yes |
| `REEMBOLSO` | Total refunds | yes |
| `GASTOS` | Total spending | yes |
| `TOTAL TURNO` | Gross turnover | yes |
| `TOTAL TRANSFERENCIA` | Total bank transfers | yes |
| `TOTAL DEPOSITO` | Total safe deposits | yes |
| `TOTAL NETO` | Net financial result | yes |

### Section: `rooms`

**Header row**: `rooms;fecha;habitacion;torre;piso;num_hab;status;inicio_servicio;fin_servicio;precio;servicio_s;extension_s;efectivo_s;consecutive_trans;refunded`

| Column | Description | Format |
|--------|-------------|--------|
| `fecha` | Change date | `yyyy/MM/dd - HH:mm:ss` |

| `torre` | Tower index | 1-based |

| `refunded` | Whether refunded | `TRUE` / `FALSE` |

### Email integration

CSV files are discoverable by `EmailConfigurationService.resolveAttachmentPaths()` when the `"Reporte CSV"` attachment checkbox is selected in the "REPORTE TURNOS" email case configuration. The resolver scans `reports/` for files matching `Turno_<turnNumber>_*.csv`.

### Affected class

| Class | Method | Role |
|-------|--------|------|
| `TurnReportGenerator` | `writeCsvReport()` | Generates CSV content from `TurnDetails` |
| `TurnReportGenerator` | `generateReport()` | Invokes `writeCsvReport()` after XLSX write |
| `EmailConfigurationService` | `resolveAttachmentPaths()` | Maps `"Reporte CSV"` → file lookup |

## 2. JSON Schema — Version 3

### Version field detection

```
optInt("version", 0):
  0 → legacy V0.1.2 (no version field)
  2 → V0.1.3 / V0.1.4
  3 → current V0.1.4.1
```

### Changes from V0.1.3 — V0.1.4

| Change | Impact | Backward compat |
|--------|--------|:---------------:|
| `"version": 3` in all files | Informational; migration v2→v3 runs automatically on deserialize | v2 files migrated |
| Tower numbers 1-based → 0-based | `RoomBookingActivity.towerNumber`, `RoomSwapActivity.*TowerNumber` decremented by 1 | Automated in `TurnDetails.migrateActivitiesIfNeeded()` |
| `roomData` field added to `RoomBookingActivity` | Standardized `RoomData` record embedded | Generated from individual fields if absent |
| `originalRoomData` / `swapRoomData` added to `RoomSwapActivity` | Standardized `RoomData` records | Generated from individual fields if absent |
| `refundRoomData` added to `RefundActivity` | Standardized room data (nullable) | Absent on legacy data |
| `roomSoldToData` added to `SaleActivity` | Standardized room data (nullable) | Absent on legacy data |
| `RoomData` record type introduced | `{ towerNumber, floorNumber, roomNumber, roomString }` | New type — no legacy equivalent |

### Migration v2 → v3

`TurnDetails.migrateActivitiesIfNeeded()` runs automatically during `TurnDetails.fromJson()`:

```
if version < 3:
  for each RoomBookingActivity:
    towerNumber = towerNumber - 1
    wrap individual fields into RoomData
  for each RoomSwapActivity:
    originalTowerNumber = originalTowerNumber - 1
    swappedTowerNumber = swappedTowerNumber - 1
    wrap individual fields into RoomData records
  set version = 3
```

### New type: `RoomData`

```java
record RoomData(int towerNumber, int floorNumber, int roomNumber, String roomString)
```

**JSON representation:**
```json
{
  "towerNumber": 0,
  "floorNumber": 0,
  "roomNumber": 0,
  "roomString": "101"
}
```

Tower numbers in `RoomData` are already 0-based (post-migration).

### File inventory (unchanged from V0.1.3)

| Directory | Contents | Since |
|-----------|----------|-------|
| `data/applicationProperties` | Motel identity, printer, transaction counter, room grid | V0.1.2 |
| `data/roomsInformation` | Room states (status, timers, durations) | V0.1.2 |
| `data/turn` | Active turn: financial totals + activity log | V0.1.2 |
| `data/inventory` | Item catalog: prices, stock | V0.1.2 |
| `history/{turn}-{timestamp}` | Completed turn archives (same schema as `data/turn`) | V0.1.2 |

All JSON files use `"version": 3` at the top level.
