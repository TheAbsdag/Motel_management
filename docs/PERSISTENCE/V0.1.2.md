# Persistence Schema — V0.1.2

**Date**: 2025 - May 2026 (commit `b73fbaf` — _"Organized and finished recovery of lost version"_)  
**Version field**: None — no file carried a schema version identifier  
**Purpose**: Initial stable schema after the recovery of a lost version. Durations stored in **hours**. No sealed activity hierarchy — turn data manipulated as raw `JSONArray`/`JSONObject` in `model.Turn`.
**Releases**: None, no release done for this version

All fields are **required** unless marked as optional.

---

## 1. `data/applicationProperties`

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `consecutiveTransaction` | `int` | yes | Auto-incrementing transaction counter |
| `motelName` | `string` | yes | Motel display name |
| `motelAddress` | `string` | yes | Motel street address |
| `motelID` | `string` | yes | Tax/business identifier (e.g. NIT) |
| `printerName` | `string` | no | Configured receipt printer service name |
| `roomsPerTower` | `array` | yes | Room grid layout (see below) |

Additional fields may be present in the file (e.g. `towers`) but are ignored on read by `ProgramConfig` and `RoomManager`.

#### `roomsPerTower[]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `towerNumber` | `int` | yes | Tower index (1-based) |
| `towerFloors` | `int` | yes | Number of floors in this tower |
| `towerRooms` | `array` | yes | Per-floor room definitions |

#### `towerRooms[]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `floor` | `int` | yes | Floor number (0-based) |
| `rooms` | `array` | yes | Room definitions on this floor |

##### `rooms[]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `roomString` | `string` | yes | Display name (e.g. `"101"`) |
| `roomFloor` | `int` | yes | Floor index |
| `roomNumber` | `int` | yes | Room index within floor |

The `customTimeData` field (per-room pricing overrides) was **declared** on the `Room` class as `RoomTime[]` but never read from JSON or used operationally.

### Example

```json
{
  "consecutiveTransaction": 1,
  "motelID": "000000",
  "motelAddress": "123 Main St",
  "motelName": "Motel",
  "roomsPerTower": [
    {
      "towerNumber": 1,
      "towerFloors": 2,
      "towerRooms": [
        {
          "floor": 0,
          "rooms": [
            { "roomString": "101", "roomFloor": 0, "roomNumber": 0 }
          ]
        }
      ]
    }
  ]
}
```

## 2. `data/roomsInformation`

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `rooms` | `array` | yes | Room state entries |

#### `rooms[]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `roomString` | `string` | yes | Display name matching `applicationProperties` |
| `towerNumber` | `int` | yes | Tower index |
| `floorNumber` | `int` | yes | Floor index |
| `roomNumber` | `int` | yes | Room index |
| `status` | `int` | yes | `1`=FREE, `2`=CLEANING, `3`=OCCUPIED |
| `startStatus` | `string` (ISO instant) | yes | When current status was entered |
| `endStatus` | `string` (ISO instant) | yes | Expected end time; **empty string `""`** when not OCCUPIED |
| `service` | `int` | yes | Base service duration in **hours** (0 for FREE/CLEANING) |
| `extension` | `int` | yes | Total extension time added in **hours** (defaults to `0`) |

**Important**: `service` and `extension` are in **hours**, not seconds. `endStatus` is an empty string (not absent/null) for FREE and CLEANING rooms.

### Example

```json
{
  "rooms": [
    {
      "roomString": "101",
      "towerNumber": 0,
      "floorNumber": 0,
      "roomNumber": 0,
      "status": 1,
      "startStatus": "2026-01-11T22:37:24.025629500-05:00[America/Bogota]",
      "endStatus": "",
      "service": 0,
      "extension": 0
    },
    {
      "roomString": "107",
      "towerNumber": 0,
      "floorNumber": 0,
      "roomNumber": 6,
      "status": 3,
      "startStatus": "2026-01-11T22:41:34.800087100-05:00[America/Bogota]",
      "endStatus": "2026-01-12T01:41:34.800087100-05:00[America/Bogota]",
      "service": 3,
      "extension": 0
    }
  ]
}
```

## 3. `data/turn`

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `turnNumber` | `long` | yes | Sequential turn identifier |
| `turnStart` | `string` (ISO ZonedDateTime) | yes | When the turn started |
| `turnEnd` | `string` (ISO ZonedDateTime) | no | When the turn ended; absent while active |
| `isTurnActive` | `boolean` | yes | `true` while turn is in progress |
| `totalTurn` | `long` | yes | Gross turnover (sum of all financial movements) |
| `totalSales` | `long` | yes | Total revenue from room bookings + item sales |
| `totalRooms` | `long` | yes | Revenue from room bookings only |
| `totalItems` | `long` | yes | Revenue from item sales only |
| `totalRefunds` | `long` | yes | Total refunds issued (lump sum — no item/room breakdown) |
| `totalSpending` | `long` | yes | Total operational expenses |
| `totalBankTransfers` | `long` | yes | Total bank transfers (positive value) |
| `totalDeposits` | `long` | yes | Total safe deposits (positive value) |
| `totalNet` | `long` | yes | Net financial result after all movements |
| `turnActivity` | `array` | yes | Chronological list of activities (see below) |

The totals are **computed** by `Turn.getDetailedTurnInformation()` and only appear after the first save. An initial empty turn file contains only `turnNumber`, `turnStart`, `isTurnActive`, and `turnActivity`.

### `turnActivity[]`

Each element has a `"changeType"` discriminator.

#### Common fields (all activities)

| Field | Type | Description |
|-------|------|-------------|
| `changeDate` | `string` (ISO ZonedDateTime) | When the activity was recorded |
| `changeType` | `string` | Discriminator — see below |

#### Type: `"room"` — Room status change

For **FREE** / **CLEANING** transitions:

| Field | Type | Description |
|-------|------|-------------|
| `roomString` | `string` | Room display name |
| `roomNumber` | `int` | Room index |
| `floorNumber` | `int` | Floor index |
| `towerNumber` | `int` | Tower index |
| `roomStatus` | `int` | New room status |
| `startStatus` | `string` (ISO instant) | When this status took effect |

For **OCCUPIED** transitions (all above +):

| Field | Type | Description |
|-------|------|-------------|
| `endStatus` | `string` (ISO instant) | Expected end time |
| `price` | `long` | Amount charged |
| `service` | `int` | Base service duration in **hours** |
| `extension` | `int` | Extra time added in **hours** |
| `servicedExtension` | `int` | Effective extension duration in **hours** |
| `consecutiveTrans` | `int` | Transaction counter for audit |
| `refunded` | `boolean` | Whether later refunded |

`consecutiveTrans` is only present on OCCUPIED transitions (FREE/CLEANING transitions use `consecutiveTransaction = 0` which is skipped).

#### Type: `"sale"` — Item sale

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `roomSoldTo` | `string` | yes | Room where items were delivered |
| `consecutiveTrans` | `int` | yes | Transaction counter |
| `register` | `array` | yes | Line items sold |

##### `register[]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `itemName` | `string` | yes | Product name |
| `itemID` | `long` | yes | Inventory item ID |
| `quantity` | `long` | yes | Quantity sold |
| `price` | `long` | yes | Total price for this line |
| `refunded` | `boolean` | no | Whether this item was later refunded |

#### Type: `"roomSwap"` — Room swap (guest move)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `originalRoom` | `string` | yes | Source room display name |
| `originalRoomNumber` | `int` | yes | Source room index |
| `originalFloorNumber` | `int` | yes | Source floor index |
| `originalTowerNumber` | `int` | yes | Source tower index |
| `swapedRoom` | `string` | yes | Destination room display name |
| `swapedRoomNumber` | `int` | yes | Destination room index |
| `swapedFloorNumber` | `int` | yes | Destination floor index |
| `swapedTowerNumber` | `int` | yes | Destination tower index |

**Note**: Field names use `swaped` (single `p`) — a typo preserved for backward compatibility in V0.1.3.

#### Type: `"refund"` — Partial or full refund

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `refundType` | `string` | yes | `"roomRefund"` or `"saleRefund"` |
| `consecutiveTrans` | `int` | yes | Transaction counter for this refund |
| `refundConsecutiveTrans` | `int` | no | Original transaction being refunded |
| `refundRoom` | `string` | yes | Associated room identifier |
| `price` | `long` | yes | Amount refunded (negative) |

When `refundType = "roomRefund"`:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `refundService` | `int` | yes | Service duration refunded in **hours** |

When `refundType = "saleRefund"`:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `itemID` | `long` | yes | Inventory item ID |
| `quantity` | `long` | yes | Quantity refunded |
| `itemName` | `string` | yes | Product name |

#### Type: `"spending"` — Operational expense

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `spendingDescription` | `string` | yes | Free-text description |
| `value` | `long` | yes | Amount (stored as negative in active turn) |
| `consecutiveTrans` | `int` | yes | Transaction counter |

#### Type: `"extraChange"` — Bank transfer / safe deposit

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `extraType` | `string` | yes | `"bankTransfer"` or `"safeDeposit"` |
| `extraChangeDescription` | `string` | yes | Free-text description |
| `value` | `long` | yes | Amount (stored as negative in active turn) |
| `consecutiveTrans` | `int` | yes | Transaction counter |

### Example (active turn with activities)

## 5. `history/{turn}-{timestamp}`

Same schema as `data/turn` (section 3), with the addition of:

- `turnEnd` always present (turn has ended)
- `isTurnActive` always `false`

### Example

```json
{
  "turnNumber": 2,
  "turnStart": "2026-01-10T19:00:00.000-05:00[America/Bogota]",
  "turnEnd": "2026-01-11T19:00:00.000-05:00[America/Bogota]",
  "isTurnActive": false,
  "turnActivity": [],
  "totalItems": 125000,
  "totalSales": 850000,
  "totalRooms": 720000,
  "totalRefunds": 15000,
  "totalSpending": 25000,
  "totalTurn": 860000,
  "totalBankTransfers": 200000,
  "totalDeposits": 50000,
  "totalNet": 625000
}
```

## Summary of V0.1.2 characteristics

| Aspect | Detail |
|--------|--------|
| Version field | **Not present** in any file |
| Room durations | `service` / `extension` / `servicedExtension` — **hours** (int) |
| Refund durations | `refundService` — **hours** (int) |
| `endStatus` for free rooms | Empty string `""` |
| RoomSwap field names | `swaped` (single `p`) — typo |
| Refund breakdown | Single `totalRefunds` — no `totalItemRefunds` / `totalRoomRefunds` |
| Turn model | `model.Turn` — turn data as raw `JSONArray`/`JSONObject` |
| Turn activity DTO | `model.dto.TurnActivityData` — class (not sealed) |
| Inventory selling cart | `Register.sellingList` — raw `JSONArray` |
