# Persistence Schema — V0.1.3

**Date**: May 2026 (commit `4b57cd48a4d6e9d775cd8ac5aa2d3dbb373b50d1`)  
**Version field**: `"version": 2` in every persistence file  
**Purpose**: Standardised schema with explicit versioning, seconds-based durations, sealed activity hierarchy, and field-level backward compatibility with V0.1.2.
**Releases**: V0.1.3 - V0.1.4

> **Note**: As of V0.1.4.1, the JSON schema version is `3` and CSV export is available alongside XLSX reports. See [V0.1.4.1](V0.1.4.1.md).

---

## Overview

- **Timezone**: `America/Bogota` — all timestamps use `ZonedDateTime` with this zone.
- **Versioning**: Every file carries `"version": 2`. The version is **informational only** — no automated migration logic exists. Backward compatibility is handled at individual field level.
- **Data directories**: `data/` (active), `backup/` (timestamped snapshots), `history/` (completed turns), `reports/` (XLSX + CSV exports).

## 3. `data/turn`

**Version**: `2`

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `version` | `int` | yes | Schema version (`2`) |
| `turnNumber` | `long` | yes | Sequential turn identifier |
| `turnStart` | `string` (ISO ZonedDateTime) | yes | When the turn started |
| `turnEnd` | `string` (ISO ZonedDateTime) | no | When the turn ended; absent while active |
| `isTurnActive` | `boolean` | yes | `true` while turn is in progress |
| `totalTurn` | `long` | yes | Gross turnover |
| `totalSales` | `long` | yes | Revenue from room bookings + item sales |
| `totalRooms` | `long` | yes | Revenue from room bookings only |
| `totalItems` | `long` | yes | Revenue from item sales only |
| `totalRefunds` | `long` | yes | Total refunds (room + item) |
| `totalItemRefunds` | `long` | no* | Refunds for item sales only |
| `totalRoomRefunds` | `long` | no* | Refunds for room bookings only |
| `totalSpending` | `long` | yes | Total operational expenses |
| `totalBankTransfers` | `long` | yes | Total bank transfers |
| `totalDeposits` | `long` | yes | Total safe deposits |
| `totalNet` | `long` | yes | Net financial result |
| `turnActivity` | `array` | yes | Chronological activity log |

_* `totalItemRefunds` and `totalRoomRefunds` are always computed internally but only appear in JSON if set. Newly saved files always include them with `totalRefunds = totalItemRefunds + totalRoomRefunds`._

### `turnActivity[]`

A polymorphic array. Each element has `"changeType"` as discriminator.

#### Common fields (all activities)

| Field | Type | Description |
|-------|------|-------------|
| `changeDate` | `string` (ISO ZonedDateTime) | When recorded |
| `changeType` | `string` | Discriminator |

#### Type: `"room"` — Room booking

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `roomString` | `string` | yes | Room display name |
| `roomNumber` | `int` | yes | Room index |
| `floorNumber` | `int` | yes | Floor index |
| `towerNumber` | `int` | yes | Tower index |
| `roomStatus` | `int` | yes | New status |
| `startStatus` | `string` (ISO instant) | yes | When status took effect |
| `endStatus` | `string` (ISO instant) | no | Expected end time (OCCUPIED only) |
| `price` | `long` | no | Amount charged (OCCUPIED only) |
| `serviceDuration` | `long` | no | Base duration in **seconds** |
| `extensionDuration` | `long` | no | Extra time in **seconds** |
| `servicedExtensionDuration` | `long` | no | Effective serviced duration in **seconds** |
| `consecutiveTrans` | `int` | no | Transaction counter |
| `refunded` | `boolean` | no | Whether later refunded |

For FREE/CLEANING transitions, only `changeDate`, `changeType`, `roomString`, `roomNumber`, `floorNumber`, `towerNumber`, `roomStatus`, and `startStatus` are present.

#### Type: `"sale"` — Item sale

| Field | Type | Required | Description |
|-------|------|----------|-------------|

## Backward Compatibility with V0.1.2

No automated migration runs on read. Individual `fromJson()` methods detect legacy formats at the field level.

### Field mapping

| Aspect | V0.1.2 | V0.1.3 | Detection |
|--------|--------|--------|-----------|
| Version | absent (`optInt` returns 0) | `"version": 2` | `optInt("version", 0)` |
| Room service duration | `"service"` (int, hours) | `"serviceDuration"` (long, seconds) | If `serviceDuration` exists → use it; else `service * 3600` |
| Room extension | `"extension"` (int, hours) | `"extensionDuration"` (long, seconds) | Same fallback logic |
| Room serviced extension | `"servicedExtension"` (int, hours) | `"servicedExtensionDuration"` (long, seconds) | Same fallback logic |
| Refund service | `"refundService"` (int, hours) | `"refundServiceDuration"` (long, seconds) | Same fallback logic |
| `endStatus` for free rooms | `""` (empty string) | absent / not set | `optString("endStatus", "")` — empty string treated as no end time |
| RoomSwap fields | `swaped*` (single p) | `swapped*` (double p) | If current key exists → use it; else fall back to legacy key |
| Refund breakdown | `totalRefunds` only | `totalRefunds` + `totalItemRefunds` + `totalRoomRefunds` | New files always include all three; old files lack the split |
| Turn model | `model.Turn` (raw JSON) | `model.turn.TurnDetails` (sealed `TurnActivity` subtypes) | Same JSON structure — no difference at file level |

### Version field detection

```
optInt("version", 0):
  0 → legacy V0.1.2 (no version field)
  2 → V0.1.3 - V0.1.4
  3 → V0.1.4.1 (current HEAD)
```

In V0.1.3 — V0.1.4, no behavior changes based on version — purely informational. V0.1.4.1 adds automated v2→v3 migration (tower number correction).

### Affected classes

| Class | Method | Field fallbacks handled |
|-------|--------|------------------------|
| `RoomBookingActivity` | `fromJson()` | `service`→`serviceDuration`, `extension`→`extensionDuration`, `servicedExtension`→`servicedExtensionDuration` |
| `RoomManager` | `restoreRoomStates()` | `service`→`serviceDuration`, `extension`→`extensionDuration` |
| `RefundActivity` | `fromJson()` | `refundService`→`refundServiceDuration` |
| `RoomSwapActivity` | `fromJson()` | `swaped*`→`swapped*` (all 4 fields) |
| `Register` | `getInventoryData()` | N/A (inventory unchanged between versions) |

## Summary of changes from V0.1.2

| Change | Impact |
|--------|--------|
| Added `"version": 2` to all 5 files | None — informational only |
| Room durations now in seconds | Backward-compatible read with hours→hours×3600 fallback |
| `endStatus` for free rooms now absent instead of `""` | Graceful empty-string detection |
| RoomSwap typo fixed (`swaped` → `swapped`) | Legacy `swaped*` still read on load |
| Refund breakdown added (`totalItemRefunds`, `totalRoomRefunds`) | Only present in newly saved files |
| `customTimeData` now writable in `applicationProperties` | Previously declared but never persisted |
| Sealed `TurnActivity` hierarchy introduced | Internal refactor — no file format change |

**Later superseded by V0.1.4.1** — version bumped to 3, tower numbers corrected to 0-based, `RoomData` records embedded, and CSV export introduced alongside XLSX reports.
